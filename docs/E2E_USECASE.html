END-TO-END USE CASE

Bukti Sinkronisasi Data Antar Modul — Aplikasi Keuangan Perumda Pasar Banjarmasin 2026

🔄 Update 20 Mei 2026 — NPD kini sepenuhnya tersinkronisasi dengan database terpusat (live data)

1. Arsitektur Data Flow

Seluruh data mengalir melalui satu database terpusat (SQLite). Setiap input pada modul manapun langsung tersinkronisasi ke seluruh laporan, termasuk NPD (Nota Pencairan Dana).

1Input Data — Jurnal, Persediaan, Piutang, Hutang, Aset, Anggaran
⬇
2Database Server — SQLite via REST API (1,260 jurnal, 573 anggaran, 119 aset)
⬇
3AppContext — State management sinkron real-time via refreshData()
⬇
4Semua Laporan — Dashboard, Buku Besar, Neraca, Laba Rugi, LRA, NPD, HPP, Arus Kas ✓ NPD Live
Perubahan 20 Mei 2026: NPD Report kini membaca data langsung dari tabel anggaran di database melalui AppContext — tidak lagi menggunakan file JSON statis. Ini berarti setiap perubahan anggaran di LRA, impor data Excel, atau input manual langsung terrefleksi di NPD secara real-time.

2. Use Case: Jurnal Umum → Laporan Keuangan

Skenario

Staf keuangan mencatat pembelian ATK senilai Rp 500.000 menggunakan kas kecil.

Step 1: Input Jurnal

  1. Buka modul Jurnal Umum
  2. Klik + Tambah Jurnal
  3. Isi data:
    • Tanggal: 2026-04-15
    • Akun Debit: 61040 - Beban Alat Tulis Kantor
    • Akun Kredit: 11101 - Kas Kecil
    • Debit: 500.000 | Kredit: 500.000
    • Keterangan: Pembelian ATK Bantex File
  4. Klik Simpan → Status: Pending
  5. Supervisor klik Approve → Status: Posted

Step 2: Verifikasi di Buku Besar

  1. Buka Buku Besar → pilih akun 61040
  2. Verifikasi: Saldo bertambah Rp 500.000 (sisi Debit)
  3. Pilih akun 11101 → Saldo berkurang Rp 500.000 (sisi Kredit)

Step 3: Otomatis Muncul di Laporan

✓ Auto-sync

✓ Auto-sync

3. Use Case: Persediaan → HPP → Laba Rugi

Skenario

Gudang menerima stok barang dan kemudian mengeluarkan barang untuk penjualan.

Step 1: Barang Masuk (Tambah Stok)

  1. Buka modul Persediaan
  2. Klik tab Barang Masuk
  3. Input: Beras 50 kg × Rp 12.000 = Rp 600.000
  4. Stok bertambah → Kartu stok terupdate

Step 2: Barang Keluar (Penjualan)

  1. Klik tab Barang Keluar
  2. Input: Beras 20 kg keluar untuk dijual
  3. Stok berkurang → HPP otomatis terhitung: 20 × Rp 12.000 = Rp 240.000

Step 3: Verifikasi Sinkronisasi

📦 Persediaan

Sisa stok: 30 kg (nilai: Rp 360.000)

✓ Auto-sync

🏭 HPP Report

HPP periode naik Rp 240.000

✓ Auto-sync

📋 Laba Rugi

Beban Pokok Penjualan naik Rp 240.000

✓ Auto-sync

✓ Auto-sync

4. Use Case: Anggaran → LRA → NPD ✓ Updated

Skenario

Divisi keuangan mengajukan pencairan dana untuk pemeliharaan gedung.

Step 1: Cek Pagu Anggaran

  1. Buka LRA (Laporan Realisasi Anggaran)
  2. Pilih tab Beban Operasional
  3. Lihat item "Pemeliharaan Bangunan Pasar": Pagu = Rp 600.000.000, Realisasi = Rp 150.000.000

Step 2: Verifikasi di NPD (Otomatis Tersinkronisasi)

  1. Buka NPD (Nota Pencairan Dana)
  2. Data NPD kini dibaca langsung dari tabel anggaran di database — sinkron real-time dengan LRA
  3. Lihat rincian: Anggaran, Akumulasi pencairan sebelumnya, Pencairan kali ini, Sisa
  4. Semua angka identik dengan yang ditampilkan di LRA karena sumber data sama

Step 3: Verifikasi Sinkronisasi

📈 LRA

Realisasi Beban Ops naik, % realisasi terupdate

✓ Auto-sync

💸 NPD Live DB

Akumulasi YTD terupdate, progress bar bergerak — data dari tabel anggaran (DB)

✓ Auto-sync

📊 Dashboard

KPI beban operasional terupdate real-time

✓ Auto-sync

📋 Anggaran

Sisa anggaran otomatis berkurang

✓ Auto-sync

9. Use Case: Update Anggaran → NPD + LRA Otomatis Update NEW

Skenario

Admin keuangan memperbarui data anggaran realisasi melalui impor Excel bulanan.

Step 1: Import Data Anggaran Baru

  1. Jalankan script node extractAllData.cjs dengan file Excel audit terbaru
  2. Data anggaran per bulan (573 item) di-seed ke tabel anggaran
  3. Setiap item berisi: kode, nama, kategori, bulan, anggaran_awal, target_bulan, sd_bln_lalu, bulan_ini, realisasi

Step 2: Verifikasi di LRA

  1. Buka LRA → Pilih periode bulan terkait
  2. Data Realisasi, Target/Bulan, dan Sd Periode Lalu otomatis terupdate
  3. Progress bar menunjukkan persentase realisasi terbaru

Step 3: Verifikasi di NPD (Otomatis)

  1. Buka NPD → Pilih periode yang sama
  2. KPI cards menunjukkan total pencairan terbaru
  3. Kategori cards (Investasi/Operasional/Umum) menunjukkan pagu & realisasi terbaru
  4. Tabel NPD menampilkan pencairan per bulan dari data anggaran live
  5. Tidak perlu import file NPD terpisah — semua data otomatis dari DB

Step 4: Verifikasi Sinkronisasi Penuh

📈 LRA

Tabel realisasi anggaran per kategori terupdate

✓ Auto-sync dari DB

💸 NPD

Pencairan dana, akumulasi YTD, dan sisa anggaran terupdate

✓ Auto-sync dari DB

📊 Dashboard

KPI beban & realisasi terupdate real-time

✓ Auto-sync

📋 Semua Laporan

Neraca, Laba Rugi, Arus Kas — semua dari sumber data terpusat

✓ Auto-sync
